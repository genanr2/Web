    Практическое руководство. Включение и запуск отладки скриптов из Internet Explorer

    Visual Studio 2013
    Другие версии
    • .NET Framework 4
    • Visual Studio 2008
    • Visual Studio 2012

    В этом разделе описываются процедуры, выполняющиеся в Internet Explorer во время отладки кода, исполняемого на стороне клиента, с помощью Visual Studio.

    Совет

    В приведенных ниже процедурах используются меню Сервис и Вид, однако содержащая их Строка меню может не отображаться. Чтобы отобразить Строку меню в Internet Explorer 9, щелкните правой кнопкой мыши строку заголовка и выберите Строка меню. Для отображения Строки меню в Internet Explorer 7 или Internet Explorer 8 щелкните кнопку Сервис, щелкните Панели инструментов, а затем выберите Строка меню.

    Включение отладки скриптов в Internet Explorer

    1. В меню Сервис выберите пункт Свойства обозревателя.

    2. В диалоговом окне Свойства обозревателя перейдите на вкладку Дополнительно.

    3. В категории Обзор снимите флажок Запретить отладку скриптов.

    4. Нажмите кнопку ОК.

    5. Выйдите и повторно запустите Internet Explorer.

    Начало отладки скриптов в Visual Studio из Internet Explorer

    1. Отладка скриптов должна быть включена, как описано в предыдущей процедуре.

    2. В меню Вид в Internet Explorer выберите Отладчик сценариев (или Внешний отладчик сценариев), а затем нажмите кнопку Открыть.

      Появится диалоговое окно JIT–отладчик Visual Studio.

    3. В списке Доступные отладчики выберите Новый экземпляр Visual Studio.

    4. Нажмите кнопку Да.

      Появляется новый экземпляр Visual Studio, и начинается отладка. В Visual Studio теперь можно открыть документы скрипта из обозревателя решений, поместить точки останова в сценарии, пройти по коду скрипта, а также просмотреть переменные и свойства в окнах переменных или окне Интерпретация.

    Начало отладки скриптов в Visual Studio со следующего оператора скрипта

    1. Отладка скриптов должна быть включена, как описано в предыдущей процедуре.

    2. В меню Вид в Internet Explorer выберите Отладчик скриптов, а затем нажмите Прервать на следующем операторе.

      Появится диалоговое окно JIT–отладчик Visual Studio.

    3. В списке Доступные отладчики в диалоговом окне JIT–отладчик Visual Studio выберите Новый экземпляр Visual Studio.

    4. Нажмите кнопку Да.

      Появляется новый экземпляр Visual Studio, но отладка не начинается до выполнения следующего оператора скрипта. Теперь можно открывать документы скрипта из обозревателя решений напрямую, но контролировать выполнение до начала отладки не получится.
